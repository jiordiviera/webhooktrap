# Webhook Trap

Welcome to the **Webhook Trap** documentation. This is your guide to capturing, inspecting, and replaying webhooks for local development and debugging.

## What is Webhook Trap?

Webhook Trap is a **self-hosted webhook debugger** for developers. It allows you to:

- **Capture** incoming webhooks from any provider (Stripe, GitHub, Shopify, etc.)
- **Inspect** raw payloads, headers, and request details
- **Replay** webhooks to localhost, staging, or production
- **Share** events with teammates via read-only links

No tunnels. No SaaS lock-in. Fully open source.

## Quick Start

### Local Docker (5 minutes)

```bash
git clone https://github.com/jiordiviera/webhooktrap.git
cd webhooktrap

docker compose up
```

Visit **http://localhost:7777** and create your first inbox.

## API Reference

Webhook Trap provides a REST API for programmatic access.

- **Authentication**: OAuth + API tokens
- **Inboxes**: Create, list, manage webhook endpoints
- **Events**: List, view, and delete captured webhooks
- **Replays**: Send webhooks to target URLs
- **Sharing**: Generate read-only share tokens

[View API Reference →](/docs/api/overview)

## Key Features

✅ **Receive webhooks** — Unique URL per inbox, always returns 200 OK  
✅ **Inspect payloads** — Raw headers, body, method, IP, timestamp  
✅ **Replay to any URL** — Localhost, staging, production  
✅ **See responses** — Status, latency, body, headers  
✅ **Share read-only links** — Show events to teammates without signup  
✅ **Anonymous mode** — No account required for first capture  
✅ **OAuth + 2FA** — GitHub/Google login, TOTP  
✅ **Self-hosted** — Full control, no vendor lock-in  

## Architecture

Webhook Trap is built with modern, production-grade tech:

- **Frontend**: Next.js 16 + React 19 + Tailwind CSS
- **Backend**: AdonisJS 6 + TypeScript
- **Database**: PostgreSQL 16
- **Storage**: Cloudflare R2 (media) or local
- **Monorepo**: pnpm + Turborepo
